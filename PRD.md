# 바이브 코딩으로 완성하는 AWS 서버리스 OpenClaw

## 발표 개요

이 세션에서는 바이브 코딩을 활용해, 최근 화제를 모으고 있는 오픈소스 프로젝트 OpenClaw를 AWS 서버리스 인프라로 마이그레이션한 Serverless-OpenClaw 프로젝트를 단 하루 만에 구축한 실전 노하우를 공유합니다.
발표자는 AWS에서 소프트웨어 개발자 및 테크니컬 트레이너로 오랜 기간 근무한 경험을 바탕으로, Fargate, Lambda, API Gateway, DynamoDB를 조합해 강력한 보안을 유지하면서도 월 $1 수준의 운영 비용을 달성한 아키텍처를 설계했습니다.
세션에서는 아키텍처 설계, 보안 강화, 비용 최적화 전략을 바이브 코딩으로 구현하는 전 과정을 다루며, 특히 바이브 코딩의 효과를 극대화하는 핵심 기법인 TDD 기반 품질 확보, 인터뷰 기반 설계, 점진적 구현, AI에게 효과적으로 맥락을 전달하는 프롬프트 전략 등 실무에서 바로 적용할 수 있는 모범사례를 소개합니다.

## 발표 내용

- OpenClaw 프로젝트의 AWS 서버리스 마이그레이션 사례
  - `~/.claude/projects/<project_name>/` 디렉토리에 기록된 OpenClaw 프로젝트 진행 로그를 분석해서 바이브 코딩으로 OpenClaw를 AWS 서버리스로 마이그레이션한 과정을 상세히 설명

- 바이브 코딩과 서버리스 아키텍처의 시너지
  - 서버리스 아키텍처가 제공하는 확장성, 비용 효율성, 관리 편의성은 누구나 인정하나, 실제로 이를 효과적으로 활용하기 위해서는 아키텍처에 대한 지식과, 개발 과정에서의 체계적인 접근이 필요
  - AWS는 수많은 서비스들을 제공하지만, 이를 어떻게 조합해서 최적의 솔루션을 구축할지는 여전히 도전적인 문제
    - 개별 서비스들도 수많은 기능과 옵션을 제공하기 때문에, 이를 효과적으로 활용하기 위해서는 깊은 이해와 경험이 필요
    - CDK와 같은 도구가 추상활를 통해 복잡도를 줄여주지만, 여전히 아키텍처 설계와 구현 과정에서 많은 세부적인 지식과 경험이 필요
      - 바이브 코딩은 마치 토니 스타크가 자비스와 함께 아이언맨 수트를 개발하는 것처럼, 개발자가 AI와 협력하여 복잡한 시스템을 설계하고 구현하는 과정을 혁신적으로 단순화해줌

- 바이브 코딩기반 서버리스 아키텍처 모범사례
  - TDD 기반 품질 확보: 서버리스 아키텍처는 분산 시스템이기 때문에, 작은 변경도 전체 시스템에 영향을 미칠 수 있음. TDD를 통해 각 컴포넌트의 품질을 보장하면서도, 전체 시스템의 안정성을 유지할 수 있음
  - 인터뷰 기반 설계: 서버리스 아키텍처는 다양한 서비스와 컴포넌트로 구성되기 때문에, 이를 효과적으로 설계하기 위해서는 명확한 요구사항과 설계 방향이 필요. 인터뷰 기반 설계를 통해 이러한 요구사항을 명확히 하고, 최적의 아키텍처를 도출할 수 있음
  - 점진적 구현: 서버리스 아키텍처는 빠르게 변화하는 요구사항에 대응하기 위해서는 점진적인 구현이 필요. 바이브 코딩은 이를 효과적으로 지원하여, 빠르게 피드백을 받고 개선할 수 있도록 도와줌
  - AI에게 효과적으로 맥락을 전달하는 프롬프트 전략: 서버리스 아키텍처는 복잡한 시스템이기 때문에, AI에게 효과적으로 맥락을 전달하는 것이 중요. 프롬프트 전략을 통해 AI가 필요한 정보를 정확히 이해하고, 적절한 솔루션을 제안할 수 있도록 도와줌
